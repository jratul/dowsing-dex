# 관동 도감 — React 개발 핸드오프 요약

1세대(관동) 포켓몬 정보·공략 사이트. 이 문서는 디자인 프로토타입(`포켓몬 도감.dc.html`)을 실제 React 프로젝트로 옮길 때의 참고 자료입니다.

---

## 1. 화면(라우트) 구성

단일 상태값 `route`로 6개 화면을 전환합니다. React에서는 React Router 등으로 분리 권장.

| route | 화면 | 핵심 내용 |
|---|---|---|
| `home` | 홈 | 히어로 + 인기 포켓몬 그리드 + 추천 공략 |
| `dex` | 도감 | 타입 필터 + 이름/번호 검색 + 카드 그리드 |
| `detail` | 포켓몬 상세 | 아트·기본정보·종족값 차트(비교)·방어 상성·진화 트리 |
| `guides` | 공략 목록 | 카테고리 탭 + 공략 카드 |
| `guide` | 공략 상세 | 본문 + 관련 포켓몬 |
| `types` | 타입 상성 | 방어 타입 계산기 + 18×18 전체 상성표 |

**추천 URL 설계**
- `/` `/pokedex` `/pokemon/:id` `/guides` `/guides/:slug` `/types`

## 2. 데이터 모델

### 포켓몬 (`P` 배열)
```ts
type Pokemon = {
  id: number;        // 전국도감 번호 (= 스프라이트 ID)
  kr: string;        // 한글 이름
  en: string;        // 영문 이름
  t: TypeName[];     // 타입 1~2개 (한글 타입명)
  s: [number,number,number,number,number,number]; // HP,공격,방어,특공,특방,스피드
  cat: string;       // 분류 (예: "화염")
  ht: number;        // 키 (m)
  wt: number;        // 몸무게 (kg)
  fl: string;        // 도감 설명
};
```
- 현재 프로토타입은 대표 55여 종만 하드코딩. **실서비스는 PokéAPI(https://pokeapi.co) 또는 자체 DB로 151종 전체 적재** 권장.
- 종족값 순서는 `[HP, 공격, 방어, 특공, 특방, 스피드]` 고정.

### 진화 체인
```ts
EVO: number[][]        // 선형 진화: [[1,2,3],[4,5,6], ...]
EEVEE: { base:133, br:[134,135,136] }  // 분기 진화(이브이)는 별도 처리
```
- 진화 트리 렌더는 "현재 포켓몬이 속한 체인"을 찾아 단계별 컬럼으로 그림. 분기형(이브이)은 1→N 구조로 예외 처리.

### 공략 (`GUIDES` 배열)
```ts
type Guide = {
  cat: '입문'|'공략'|'대전'|'포획'|'진화';
  title: string;
  icon: number;        // 썸네일로 쓸 포켓몬 ID
  rel?: number[];      // 관련 포켓몬 ID
  body?: string[];     // 문단 배열
};
```

## 3. 타입 상성 로직 (가장 재사용 가치 높음)

18타입 기준. 데이터는 **공격 타입 → {효과 굉장(s), 효과 별로(w), 무효(n)}** 형태.

```js
CHART = {
  불꽃: { s:['풀','얼음','벌레','강철'], w:['불꽃','물','바위','드래곤'], n:[] },
  // ... 18종
};

// 단일 배율
function mult(atk, def) {
  const r = CHART[atk];
  if (r.n.includes(def)) return 0;
  if (r.s.includes(def)) return 2;
  if (r.w.includes(def)) return 0.5;
  return 1;
}

// 복합 타입 방어 프로파일: 각 공격 타입의 누적 배율 (×0 ~ ×4)
function profile(defTypes) {
  return TYPE_ORDER.map(atk => ({
    type: atk,
    m: defTypes.reduce((acc, d) => acc * mult(atk, d), 1)
  }));
}
```
- **상세 화면의 약점/반감**과 **타입 계산기**가 모두 `profile()` 하나를 공유합니다.
- 약점 = `m > 1` (×2, ×4) / 반감 = `0 < m < 1` (×½, ×¼) / 무효 = `m === 0`.

### 타입 메타
- `TYPE_ORDER`: 18타입 정렬 순서
- `COLOR`: 타입별 대표 색 (배지·카드 상단 바·차트에 사용)
- `LIGHT`: 밝은 배경 타입(전기·얼음·강철·페어리·땅) → 글자색을 어둡게

## 4. 컴포넌트 분리 제안

```
<App>                      // 라우팅
 ├─ <SiteHeader>           // 데스크탑 nav / 모바일 검색 헤더
 ├─ <MobileTabBar>         // 모바일 하단 탭
 ├─ <HomePage>
 │   ├─ <Hero>
 │   ├─ <PokemonCard> ×N
 │   └─ <GuideCard> ×N
 ├─ <PokedexPage>
 │   ├─ <TypeFilter>
 │   └─ <PokemonCard> ×N
 ├─ <PokemonDetail>
 │   ├─ <StatChart>        // 비교 기능 포함
 │   ├─ <TypeDefense>      // profile() 결과 표시
 │   └─ <EvolutionTree>
 ├─ <GuideListPage> / <GuideDetailPage>
 └─ <TypeChartPage>
     ├─ <TypeCalculator>   // profile()
     └─ <TypeChartGrid>    // 18×18
```
- `<PokemonCard>`, 타입 배지(`TypeBadge`)는 전역 재사용 컴포넌트로.
- `profile()`/`mult()`는 `lib/typeChart.ts` 유틸로 분리.

## 6. 반응형

- 프로토타입은 JS로 `window.innerWidth < 720` 감지 → 모바일 전환.
- **React에서는 CSS 미디어쿼리/컨테이너 쿼리로 교체** 권장.
- 모바일 차이: 상단 nav → 하단 탭바 / 도감·인기 그리드 2열 / 상세 1열 스택 / 타입 칩·상성표 가로 스크롤 유지.
